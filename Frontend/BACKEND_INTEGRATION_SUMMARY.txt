───────────────────────────────────────────────────────────────────────────────
DATA FLOW EXAMPLES
───────────────────────────────────────────────────────────────────────────────

EXAMPLE 1: NEW USER REGISTRATION → DASHBOARD
─────────────────────────────────────────────────────────────────────────────
1. User visits /register
2. Fills form → Calls POST /auth/register
3. Backend creates user account
4. Frontend stores token in cookie + Zustand
5. Redirects to /onboarding
6. User completes questionnaire
7. Calls POST /auth/onboarding with preferences
8. Backend saves preferences
9. Redirects to /dashboard
10. Calls GET /recommendations/personalized & /trending
11. Dashboard loads with personalized recommendations

EXAMPLE 2: BROWSE PRODUCTS
─────────────────────────────────────────────────────────────────────────────
1. User navigates to /products
2. Frontend calls GET /products (default, first page)
3. Backend returns paginated products
4. Products render in grid
5. User filters by price: $10-$100
6. Calls GET /products?min_price=10&max_price=100
7. Backend returns filtered results
8. Grid updates with new products
9. User hovers over product
   → Calls POST /interactions/track {type: "product_view"}
10. User clicks product
    → Calls POST /interactions/track {type: "product_click"}
    → Navigates to /products/{id}
11. Detail page loads via GET /products/{id}
12. Backend returns full product details + reviews

EXAMPLE 3: ADD TO CART → CHECKOUT
─────────────────────────────────────────────────────────────────────────────
1. User clicks "Add to Cart" on product
2. Calls POST /cart/add {product_id, quantity: 1}
3. Backend adds to user's cart
4. Toast notification: "Added to cart"
5. Cart badge in navbar updates
6. User navigates to cart
   → Drawer shows items
7. User increases quantity
   → Calls PUT /cart/items/{id} {quantity: 2}
8. User removes item
   → Calls DELETE /cart/items/{id}
9. User navigates to /checkout
10. Page loads cart via Zustand store (local cache)
11. Shows order summary
12. User clicks "Place Order"
    → Simulates payment (2 seconds)
    → Shows confirmation
    → **(TODO)** Calls POST /orders/create
    → Calls DELETE /cart to clear
    → Calls POST /interactions/track {type: "purchase"}
13. Redirects to success screen

EXAMPLE 4: PERSONALIZED RECOMMENDATIONS
─────────────────────────────────────────────────────────────────────────────
1. User logs in
2. Frontend has onboarding_completed = true
3. Dashboard page loads
4. Calls GET /recommendations/personalized {limit: 10}
5. Backend uses:
   - User's selected categories
   - Budget range
   - Shopping interests
   - Past view history
   - Collaborative filtering
6. Returns 10 AI-selected products
7. Frontend displays in "Recommended For You" rail
8. Also calls GET /recommendations/trending {limit: 10}
9. Returns trending products for "Trending Now" section
10. Both sections render with recommendation cards
11. Cards show match score and reason badge
